# プロジェクト要件定義書

## プロジェクト概要
- プロジェクト名: 2.5vStream Atlier るりのね
- 目的: ストリーマーのためのコミュニティWebサービス
- 開始日: 2025-11-15

## コミュニティ概要
### 基本情報
- コミュニティ名: 「2.5vStream Atlier るりのね」
- 対象: 夢を持っているライバー/ストリーマー
- 運営形態: コミュニティベース
- 参加方法: ストリーマー自身が応募してコミュニティに参加

### コンセプト
- 「未だ何者にもなれないぼくらが*瑠璃*(るり　宝石)となっていく音をお届けします」
- 夢を持っている方々がライバーとして活動するコミュニティ

### 世界観設定
- 舞台: 仮想世界「虚世（uturo-yo)」
- 設定: 思念や意識、概念が漂い形となる仮想世界

#### ストリーマーの存在性
- 本質: 元々虚世を漂う思念や概念が強い思念によって固まった存在
- 活動原理: 思念を核として（夢）活動する
- 多様性: 現実世界、異世界、空想など様々な起源を持つため統一性は無い
- 共通性: 各自の思念の核で動くことのみが共通の行動原理

#### 虚世の構造
- 虚都（kyoto）: 人格を持たない思念が建物や町として形作ったもの
- 多層的世界: 様々な思念が混在し形を成している

#### 現実世界との接続
- 電子の力: ネットを通じて現実世界と繋がりを持つ
- ロボット媒体: ロボットを媒体として現実世界に存在することが可能

### 現状
- コミュニティはまだ発足していない
- Webサービスの開始と共に活動開始予定

## 機能要件

### ストリーマー応募プロセス
#### 募集方法
- Discord（ストリーマーのコミュニティサーバ、るりのねのコミュニティサーバ）
- 配信時の告知

#### 応募フォーム
- Googleフォームを活用
- 必要情報の入力

#### 審査プロセス
- 運営側での審査実施
- 承認後のWebサービス用アカウント払い出し
- 各種連絡先への送付

#### 審査基準（すべてを掲示はしない）
- 過去に問題を犯していない（ネット調査実施、調査方法は非掲載）
- 反社および反社関係者でない（掲載）
- 16歳以上（高校生の場合、保護者の承認必須）（掲載）

#### 除隊処分条件
- 審査基準に乖離する行動
- コミュニティメンバーとのトラブル
- 荒らし、暴言、コミュニティの風紀を乱す行為
- コミュニティ提供サービスの第三者への共有

### 特典提供（段階的展開）
#### Phase01: サービス開始時
- **Discordサーバの提供**（問い合わせ・ナレッジ蓄積のみ）
- **配信補助ツールの提供**（リリース後Webサービスに追加）
  - ルーレット
  - ビンゴ
  - パネル開け
  - 週次スケジュール告知画像作成
- **ナレッジ投稿機能**
- **母体サービスの案内**
  - O-EN（現在準備中）※別途詳細契約が必要
  - 機材購入相談・特価提供 ※別途詳細契約が必要
- **コミュニティコンセプトの利用許可**
  - コミュニティオリジナルサムネイルの利用許可
  - コミュニティマークの利用許可

#### Phase02: コミュニティメンバー100人達成
- **事務所機能の提供** ※別途詳細契約が必要
- **Discordコミュニティサーバの交流チャンネル開始**

#### Phase03: コミュニティXフォロワー1000人達成 & 運用資金200万円確保
- **オーディション開催**
- **選抜メンバー選定開始**

## 技術要件

### 基本方針
- 基本的に無料で環境を構築
- セキュリティのため認証方式のみ有料環境を活用

### 技術スタック
- **フロントエンド**: CloudFlareのAstro Framework Starter（既にwww配下にテンプレート展開済み）
- **バックエンド**: CloudFlare Workers
- **ホスティング**: CloudFlare
- **認証方式**: AWS IAM（他に最適なプロダクトがあれば変更検討）
- **ソースコード管理**: GitHub

### 対応環境
- PC、mobile両対応
- SEO最適化
- 更新が容易な状態

### プロジェクトパラメータ
- **ドメイン**: rurinone.ink
- **WebサービスURL**: 
  - Public: https://www.rurinone.ink
  - Community: https://community.rurinone.ink
  - Operate: https://manage.rurinone.ink
- **CloudFlareデフォルトURL**: rurinone.d-akame.workers.dev
- **GitHubリポジトリ**: https://github.com/AkameLLC/rurinone (モノレポ構成)

## 非機能要件

### パフォーマンス要件
- **想定ユーザー数**: 最大500ユーザー
- **パフォーマンス**: 特別な要件なし（非同期で更新されるようにしたい）

### セキュリティ要件
- 個人情報をIAMにて管理
- 機密漏洩しないようにしたい
- コスト見合いでの検討が必要

### 可用性・保守性要件
- 特別な要件なし

### 配信補助ツール要件
- **各ツール詳細仕様**: 現時点で検討不要
  - ルーレット
  - ビンゴ  
  - パネル開け
  - スケジュール画像作成
- **配信状態取得**: APIにて取得予定

## 制約事項

### 著作権・権利関係
- **配信データおよび当コミュニティ以外のデータ**: 既存の帰属元に権利が帰属
- **当コミュニティのツール・画像データ等**: コミュニティに権利が帰属
- **脱退時の義務**: 当コミュニティにて配布した画像・データ・設定を削除する

## Webサイト構成設計

### サイト全体構造（サブドメイン構成）
```
Public (https://www.rurinone.ink/):
├── / (トップページ)
├── /about (コミュニティ説明)
├── /streamers (配信者一覧)
├── /streamers/[id] (配信者詳細)
├── /news (ニュース)
├── /news/[slug] (ニュース詳細)
├── /apply (応募フォーム)
├── /contact (問い合わせ)
├── /company (母体会社説明)
├── /login (ログイン)
└── /terms (利用規約)

Community (https://community.rurinone.ink/):
├── / (ダッシュボード)
├── /tools (配信補助ツール)
├── /knowledge (ナレッジ)
├── /services (母体サービス案内)
└── /assets (コミュニティ素材)

Operate (https://manage.rurinone.ink/):
├── / (管理ダッシュボード)
├── /users (ユーザー管理)
├── /streamers (配信者管理)
├── /news (ニュース管理)
└── /settings (システム設定)
```

### ページレイアウト構成
```
[フルスクリーンキービジュアル] ← 別途準備
[Header/Navigation]
[メインコンテンツ]
[Footer]
```

### 主要コンポーネント設計
1. **キービジュアル**
   - フルスクリーン表示
   - 虚世（uturo-yo）の世界観を表現
   - ヘッダーより上部に配置

2. **Header/Navigation**
   - ログイン状態に応じたメニュー表示
   - 虚世の世界観を表現するデザイン

3. **Footer**
   - 利用規約、プライバシーポリシーへのリンク
   - コミュニティSNSリンク

4. **配信者カード**
   - 配信状態の表示
   - プロフィール情報
   - X、配信媒体へのリンク

5. **認証フロー**
   - AWS IAMとの連携

6. **ダッシュボード**
   - Phase01特典への導線
   - ツール、ナレッジ、サービス案内の整理
